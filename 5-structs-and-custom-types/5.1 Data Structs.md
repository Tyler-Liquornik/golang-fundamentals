Let's start going beyond values types, to group data and functions into collections with a new type called the **Struct**. They key difference between Java classes and Go structs is that Go is not an OOP language, so Go structs use *composition*, not *inheritance*. 

Consider a program that collects several data points from the user. We probably want to group that data together into a struct to be able to pass that around to other functions to process the data. 

`structs.go`:
```go
package main

import (
	"fmt"
)

func main() {
	firstName := getUserData("Please enter your first name: ")
	lastName := getUserData("Please enter your last name: ")
	birthDate := getUserData("Please enter your birth date (MM/DD/YYYY): ")

	// ... do something with the gathered data

	fmt.Println(firstName, lastName, birthDate)
}

func getUserData(promptText string) string {
	fmt.Print(promptText)
	var value string
	fmt.Scan(&value)
	return value
}
```

Let's start with a demo of a simple struct that just acts as a container for some data. 

We need both the `type` and `struct` keywords to create a struct like `type user struct` because `type` declares a new named type, and `struct` defines what kind of type it is (there are other types, beside `struct`, we'll get to that).

`structs.go`:
```go
package main

import (
	"fmt"
	"time"
)

type user struct {
	firstName string
	lastName  string
	birthDate string
	createdAt time.Time
}

func main() {
	firstName := getUserData("Please enter your first name: ")
	lastName := getUserData("Please enter your last name: ")
	birthDate := getUserData("Please enter your birth date (MM/DD/YYYY): ")

	// ... do something awesome with that gathered data!

	fmt.Println(firstName, lastName, birthDate)
}

func getUserData(promptText string) string {
	fmt.Print(promptText)
	var value string
	fmt.Scan(&value)
	return value
}
```

> Just like functions, whether or not the type is capitalized defines the access control of the type. `user` is private, `User` would be public.

> We also used `time.Time` here to get the time the data was created (a common pattern we know well in backend engineering). This is an example of a using a struct exposed by another package, in this case the standard library package `time`.

Now lets instantiate a struct. We typically create structs using **struct (composite) literals**, since Go doesn’t have built-in constructor syntax like Java, although you can optionally add a custom function to act as a constructor.

`structs.go`:
```go
package main

import (
	"fmt"
	"time"
)

type user struct {
	firstName string
	lastName  string
	birthDate string
	createdAt time.Time
}

func main() {
	firstName := getUserData("Please enter your first name: ")
	lastName := getUserData("Please enter your last name: ")
	birthDate := getUserData("Please enter your birth date (MM/DD/YYYY): ")

	var appUser = user{
		firstName: firstName,
		lastName:  lastName,
		birthDate: birthDate,
		createdAt: time.Now(),
	}

	outputUser(appUser)
}

func outputUser(u user) {
	fmt.Println(u.firstName, u.lastName, u.birthDate)
}

func getUserData(promptText string) string {
	fmt.Print(promptText)
	var value string
	fmt.Scan(&value)
	return value
}
```

> If you intend on passing a value for every field, in the same order they are listed in the struct declaration, you can omit they keys and just pass the values. For example, `appUser = user{firstName, lastName, birthDate, time.Now()}`

> Any omitted fields will instantiate will default values, just like Java. You can have a fully empty struct like `appUser = user{}`, just like an empty object in Java `appUser = new User()`.
