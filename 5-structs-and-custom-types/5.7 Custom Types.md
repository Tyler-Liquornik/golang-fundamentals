
Recall that to create a struct `x`, we write `type x struct`. So a `struct` is a kind of type. There are however other ways to use `type`.

One such way is to use `type` for aliasing other types. We could alias `string` to `str`.

```go
package main

import "fmt"

type str = string

func main() {
	var name str
	name = "Tyler"
	fmt.Print(name)
}
```

Well, this is kind of helpful... like bash aliasing, we can save some characters with type aliasing.

What is more helpful is that we can define `str` as a new type based on `string`, which is different from aliasing. This unlocks the ability to add a *receiver method* to the `str` type. 

```go
package main

import "fmt"

type str string

func (text str) log() {
	fmt.Println(text)
}

func main() {
	var name str
	name = "Tyler"
	name.log()
}
```

> You could also have wrapped the value type in a struct (e.g., `struct { str string }`) and attach methods to that struct, but defining a new type like type `str string` is cleaner and the preferred approach when you just want to extend a primitive without adding extra structure.
