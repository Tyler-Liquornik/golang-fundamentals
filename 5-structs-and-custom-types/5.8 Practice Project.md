Let's build a practice project, where we take in user input for a note, and then save that note to a JSON file. 

We'll explore some new standard library packages along the way
- `bufio` provides buffered I/O, commonly used for reading input line-by-line, even with whitespaces.
- `strings` for string manipulation
- `encoding/json` for JSON encoding/decoding

`note.go`:
```go
package note

import (
	"encoding/json"
	"errors"
	"os"
	"strings"
	"time"
)

type Note struct {
	Title     string
	Content   string
	CreatedAt time.Time
}

func New(title, content string) (Note, error) {
	if title == "" || content == "" {
		return Note{}, errors.New("title and content cannot be empty")
	}

	return Note{
		title,
		content,
		time.Now(),
	}, nil
}

func (n Note) String() string {
	return "Note title: " + n.Title + "\nNote content: " + n.Content + "\nNote created at: " + n.CreatedAt.String()
}

func (n Note) Save() error {
	fileName := strings.ToLower(strings.ReplaceAll(n.Title, " ", "_")) + ".json"
	filePathWithName := "5-structs-and-custom-types/5.8/" + fileName

	// Note: json.Marshal() will only marshal the exported fields of the struct,
	// so all fields have been made public.
	jsonBytes, err := json.Marshal(n)
	if err != nil {
		return err
	}
	return os.WriteFile(filePathWithName, jsonBytes, 0644)
}

```

`main.go`:
```go
package main

import (
	"bufio"
	"fmt"
	"os"
	"strings"

	"github.com/Tyler-Liquornik/golang-fundamentals/5-structs-and-custom-types/5.8/note"
)

func main() {
	title, content := getNoteData()
	userNote, err := note.New(title, content)
	if err != nil {
		fmt.Println("Error creating note instance:", err)
		return
	}

	noteString := userNote.String()
	fmt.Println(noteString)

	err = userNote.Save()
	if err != nil {
		fmt.Println("Error saving note:", err)
		return
	}
	fmt.Println("Note saved successfully!")
}

func getNoteData() (string, string) {
	title := getUserInput("Enter note title: ")
	content := getUserInput("Enter note content: ")
	return title, content
}

func getUserInput(prompt string) string {
	fmt.Print(prompt)

	// Use bufio to read the entire line of input including spaces,
	// unlike fmt.Scan/Scanln which only read whitespace-separated tokens.
	reader := bufio.NewReader(os.Stdin)
	text, err := reader.ReadString('\n')
	if err != nil {
		fmt.Println("Error reading input, please try again.")
		return getUserInput(prompt)
	}

	// Remove newline characters added when the user presses Enter:
	// "\n" for Unix/macOS, "\r\n" for Windows
	text = strings.TrimSuffix(text, "\n")
	text = strings.TrimSuffix(text, "\r")
	return text
}
```

Now when if we run the program and enter some text:

```sh
Enter note title: My Note
Enter note content: Hello World!
Note title: My Note
Note content: Hello World!
Note created at: 2026-04-09 13:58:24.964673 -0400 EDT m=+5.821684668
Note saved successfully!
```

We get a JSON file saved for our note:

`my_note.json`
```json
{"Title":"My Note","Content":"Hello World!","CreatedAt":"2026-04-09T13:58:24.964673-04:00"}
```

There is one thing we might want to change about this program though. It is unusual to have JSON keys capitalized. To solve this, we can use **struct tags**. Struct tags add metadata to struct fields. This is similar to annotations in Java or decorators in Python. In this case, `encoding/json.Marshal()` uses this metadata for custom field key names if present. 

We add struct tag metadata using a string in backticks after the field type. In our case, to work with `encoding/json.Marshal()`:

`note.go` (snippet):
```go
type Note struct {  
    Title     string    `json:"title"`  
    Content   string    `json:"content"`  
    CreatedAt time.Time `json:"created_at"`  
}
```

And now we get a cleaner JSON output:

`my_note.json`:
```
{"title":"My Note","content":"Hello World!","created_at":"2026-04-09T14:07:10.972394-04:00"}
```

> Struct tags only do anything when a package reads and interprets them. Adding any struct tags by themselves doesn't do anything.
