Consider we want to format our text nicely. Right now, we are outputting dollar values with too many decimals. Let's round them to 2 decimals.

`investment_calculator.go`:
```go
package main

import (
	"fmt"
	"math"
)

func main() {
	const inflationRate = 2.5

	var investmentAmount, years, expectedReturnRate float64

	fmt.Print("Enter an investment amount: ")
	fmt.Scan(&investmentAmount)

	fmt.Print("Enter a number of years: ")
	fmt.Scan(&years)

	fmt.Print("Enter an expected return rate: ")
	fmt.Scan(&expectedReturnRate)

	futureValue := investmentAmount * math.Pow(1+expectedReturnRate/100, years)

	futureValueInflationAdj := futureValue / math.Pow(1+inflationRate/100, years)
	
	fmt.Printf("Future Value: %.2f\nFuture Value (adjusted for inflation): %.2f", futureValue, futureValueInflationAdj)
}
```

`Printf` has the following placeholders available:
- `%v` default format for any value
- `%T` type of the value
- `%d` integer (base 10)
- `%f` floating point
- `%.nf` float with n decimal places
- `%n.nf` n width and n decimal places
- `%-n.nf` n left-aligned width and n decimal places
- `%e` scientific notation
- `%s` string
- `%q` quoted string
- `%c` character (rune)
- `%t` true / false
- `%p` memory address

> `Println` can also take many values to print all of them in one line, whitespace delimited `fmt.Println("Future Value:", futureValueInflationAdj)`

> `fmt` also provides “S” versions of `Print`, `Printf`, and `Println` to store the resulting strings instead of printing them to standard output. Functions like `fmt.Sprint`, `fmt.Sprintf`, and `fmt.Sprintln` return the formatted string, which you can then assign to a variable, or use elsewhere in your program. This is similar to `String.format` in Java.

> Go also supports multiline strings using backticks (raw string literals). These preserve line breaks and treat escape sequences like `\n` or `\t` as literal text, rather than interpreting them.
