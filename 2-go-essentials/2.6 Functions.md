Let's split off some code into a separate function:

`investment_calculator.go`:
```go
package main

import (
	"fmt"
	"math"
)

const inflationRate = 2.5

func main() {

	var investmentAmount, years, expectedReturnRate float64

	outputText("Enter an investment amount: ")
	fmt.Scan(&investmentAmount)

	outputText("Enter a number of years: ")
	fmt.Scan(&years)

	outputText("Enter an expected return rate: ")
	fmt.Scan(&expectedReturnRate)

	futureValue, futureValueInflationAdj := calculateFutureValues(investmentAmount, expectedReturnRate, years)

	fmt.Printf("Future Value: %.2f\nFuture Value (adjusted for inflation): %.2f", futureValue, futureValueInflationAdj)
}

func outputText(text string) {
	fmt.Print(text)
}

func calculateFutureValues(investmentAmount, expectedReturnRate, years float64) (fv float64, fvAdj float64) {
	fv = investmentAmount * math.Pow(1+expectedReturnRate/100, years)
	fvAdj = fv / math.Pow(1+inflationRate/100, years)
	return
}
```

Like Java, Go is statically typed, so function parameters and return values must have defined types. However, if a function does not return anything, Go simply omits the return type instead of using a keyword like `void`. Go also can easily return multiple values without wrapping them in a list, by comma delimiting values to return in the `return` statement.

For function input parameters, a notation like `investmentAmount, expectedReturnRate, years float64` is a shorthand when all input parameters have the same type, so thats syntactic sugar for `investmentAmount float64, expectedReturnRate float64, years float64`

For function return values, Go supports a feature called **named return values**, where you declare the return variables in the function signature (e.g., `(fv float64, fvAdj float64)`). These variables are initialized to their default values and can be assigned within the function body without needing `:=`, since they are already declared. You still need to assign them for the function to be useful. Since we also know we are returning those exact values, we can omit them from the return statement too and just write `return`.
