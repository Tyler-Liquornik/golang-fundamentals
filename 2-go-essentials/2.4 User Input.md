Now let's collect user input for the custom values that might vary.

`investment_calculator.go`:
```go
package main

import (
	"fmt"
	"math"
)

func main() {
	const inflationRate = 2.5

	var investmentAmount, years, expectedReturnRate float64

	fmt.Print("Enter an investment amount: ")
	fmt.Scan(&investmentAmount)

	fmt.Print("Enter a number of years: ")
	fmt.Scan(&years)

	fmt.Print("Enter an expected return rate: ")
	fmt.Scan(&expectedReturnRate)

	futureValue := investmentAmount * math.Pow(1+expectedReturnRate/100, years)

	futureValueInflationAdj := futureValue / math.Pow(1+inflationRate/100, years)

	fmt.Println(futureValueInflationAdj)
}
```

> In Go, `fmt.Scan` takes a simpler approach by writing directly into variables by taking their memory address, unlike Java's `Scanner` class, which returns values (e.g., `int x = scanner.nextInt()`). This is why you must pass the **address** of a variable using `&x`, you’re giving `fmt.Scan` access to the actual memory location so it can modify the original value, rather than a copy. We’ll dive deeper into pointers later.

> Like Java, declaring variables without assigning a value gives them default values, such as 0.0 for `float64`, 0 for `int`, false for `bool`, and "" for `string`, while reference types like pointers, slices, and maps default to `nil`.
