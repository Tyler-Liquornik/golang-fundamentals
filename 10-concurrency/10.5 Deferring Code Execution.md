Manually calling `Close` to close the file in `filemanager.go` is to use a special keyword `defer` to defer the file close operation. `defer` will defer the schedule the code to run after the current function has finished running. This way we won't have to remember to do it later manually and it will always get done whether file I/O functions return successfully or with an error. `defer` is somewhat similar to `finally` in Java.

We can write `defer file.Close()` as soon as we know the file opened successfully when reading from or writing to files.

`filemanager.go` (snippet):
```go
func (fm FileManager) ReadLines() ([]string, error) {
	file, err := os.Open(fm.InputFilePath)
	if err != nil {
		fmt.Println("Error opening file:", err)
		return nil, err
	}

	defer file.Close()

	scanner := bufio.NewScanner(file)
	var lines []string

	for scanner.Scan() {
		lines = append(lines, scanner.Text())
	}
	err = scanner.Err()
	if err != nil {
		fmt.Println("Error reading file:", err)
		return lines, err
	}

	return lines, nil
}

func (fm FileManager) WriteResult(data any) error {  
    file, err := os.Create(fm.OutputFilePath)  
    if err != nil {  
       fmt.Println("Error creating file:", err)  
       return errors.New("failed to create file")  
    }  
  
    defer file.Close()  
  
    time.Sleep(3 * time.Second)  
  
    encoder := json.NewEncoder(file)  
    err = encoder.Encode(data)  
    if err != nil {  
       fmt.Println("Error writing JSON data:", err)  
       return errors.New("failed to write JSON data")  
    }  
      
    return nil  
}
```

> There can be many defer statements in a function, each applying to a single function call. They run in LIFO order, meaning the most recently deferred call executes first.
