Let's start with:

`main.go`:
```go
package main

import (
	"fmt"
	"time"
)

func greet(phrase string) {
	fmt.Println("Hello!", phrase)
}

func slowGreet(phrase string) {
	time.Sleep(3 * time.Second) // simulate a slow, long-taking task
	fmt.Println("Hello!", phrase)
}

func main() {
	greet("Nice to meet you!")
	greet("How are you?")
	slowGreet("How ... are ... you ...?")
	greet("I hope you're liking the course!")
}
```

Result:
```sh
Hello! Nice to meet you!
Hello! How are you? ## 3 seconds of pause
Hello! How ... are ... you ...?
Hello! I hope you're liking the course!
```

Consider we want to run these functions concurrently so that `slowGreet` doesn't block thread execution. We can create **goroutines** by simply using the `go` keyword to run multiple function calls in parallel.

We might first try:

`main.go` (sinppet):
```go
func main() {
	go greet("Nice to meet you!")
	go greet("How are you?")
	go slowGreet("How ... are ... you ...?")
	go greet("I hope you're liking the course!")
}
```

This basically is dispatching 4 different goroutines. But in the end nothing is printed because the main function ends before anything can print, as soon as the goroutines are all dispatched.
