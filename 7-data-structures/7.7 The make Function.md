Consider the code:

`main.go`:
```go
package main

import "fmt"

func main() {
	userNames := []string{}
	userNames = append(userNames, "Tyler")
	userNames = append(userNames, "Jordan")

	fmt.Println(userNames)
}
```

It's kind of inefficient to have go first create an empty array under the hood, then grow it as we append more elements. It would be better if we could just the underlying array be the right size right away. We can do this with the built-in `func make(T, sizeParams ...int) T` where `T` is a type literal that is ether a slice, map, or channel (we'll get to channels later).  `sizeParams` for a slice is 2 values for `len` and `cap`, or you can exclude `cap` and it gets assigned the value of `len`. 

`main.go`:
```go
package main

import "fmt"

func main() {
	userNames := make([]string, 2)
	userNames = append(userNames, "Tyler")
	userNames = append(userNames, "Jordan")

	fmt.Println(userNames)
}
```

Result:
```sh
[  Tyler Jordan]
```

> Notice the empty spaces are at the front. That's cause `len = 2`, so those empty strings `""` "count" as entries and we append after them. > Remember it is `len` that determines where `append` adds the next element. If we want to append at index 0, we'd set `len = 0`. `cap` is what determines how many contiguous memory blocks to allocate for the underlying array.

We can use `make` for the same efficiency gains for a `map`. To `make` a `map`, `sizeParams` needs 1 argument for the size of the map.

`main.go`:
```go
package main

import "fmt"

func main() {
	userNames := make([]string, 2)
	userNames = append(userNames, "Tyler")
	userNames = append(userNames, "Jordan")

	fmt.Println(userNames)

	courseRatings := make(map[string]float64, 2)
	courseRatings["Go"] = 4.7
	courseRatings["React"] = 4.5

	fmt.Println(courseRatings)
}
```

In summary, if you know (or even know roughly) how much memory you'll need, it's best to pre-allocate. Sometimes you know, sometimes you don't, thats how you choose `make` vs a literal to create array-based data structures.
