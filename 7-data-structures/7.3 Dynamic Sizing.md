Slices have another feature we haven't shown yet that makes them enormously useful, which is that they provide dynamic sizing. Using the built-in `append` function, you can add element the one index after the capacity. When this happens, Go allocates a new, larger underlying array and copies the existing elements over to handle dynamic sizing, similar to how `ArrayList<T>` works under the hood in Java.

> `append` returns a new slice, so you should always assign the result. It may reuse the same underlying array (no new memory) if there’s enough capacity, or allocate a new array and copy the elements if not.

`lists.go`:
```go
package main

import "fmt"

func main() {
	prices := []float64{10.99, 8.99}
	updatedPrices := append(prices, 12.99)

	fmt.Println(prices)
	fmt.Println(updatedPrices)

	arr := [5]float64{10.99, 8.99}
	pricesWithExtraCap := arr[:2]
	pricesWithExtraCap = append(pricesWithExtraCap, 12.99)

	fmt.Println(arr)
	fmt.Println(pricesWithExtraCap)
}
```

Result:
```sh
[10.99 8.99]
[10.99 8.99 12.99]
[10.99 8.99 12.99 0 0] # These 0's are extra cap not part of len because we never epxplicit wrote them out like [5]float64{10.99, 8.99, 0, 0, 0}
[10.99 8.99 12.99]
```
