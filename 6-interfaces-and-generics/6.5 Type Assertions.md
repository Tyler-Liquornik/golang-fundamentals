Something like `p.(type)` only works for type switching like `switch p.(type) { ... }`. There is another similar syntax for **type assertions**, that works anywhere, not just in `switch` blocks. To do a type assertion to assert `p` is of type `T`, we write `p.(T)`. > Note that this only works if `p` is an interface value, type assertions cannot be used on non-interface types.

`main.go`:
```go
package main

import "fmt"

type Printer interface {
	Print()
}

type Person struct {
	Name string
}

func (p *Person) Print() {
	fmt.Println("Name:", p.Name)
}

func callPrint(p Printer) {
	switch concreteValue := p.(type) {
	case *Person:
		if concreteValue == nil {
			fmt.Println("Cannot print: Person is nil")
			return
		}
		concreteValue.Print()
	default:
		p.Print()
	}
}

func tryPerson(p Printer) {
	typedValue, ok := p.(*Person)

	if !ok {
		fmt.Println("Not a Person")
		return
	}

	if typedValue == nil {
		fmt.Println("Person is nil")
		return
	}

	person.Print()
}

func main() {
	validPerson := &Person{Name: "Tyler"}

	fmt.Println("=== Type Switch ===")
	callPrint(validPerson) // Works

	var nilPerson *Person = nil
	callPrint(nilPerson) // Works, now Handled safely

	fmt.Println("\n=== Type Assertion ===")
	var p Printer = nil
	
	tryPerson(validPerson)
	tryPerson(nilPerson)
	tryPerson(p) // Not a Person
}
```

> A type assertion `p.(T)` returns two values: the value cast to type `T`, and a boolean (idiomatically called `ok`)indicating whether the assertion succeeded. If the assertion fails (i.e., `ok == false`), the returned value is the default value of type `T`, often `nil`. Even if the assertion succeeds, the underlying value might still be nil, so both the boolean and the value must be checked.
