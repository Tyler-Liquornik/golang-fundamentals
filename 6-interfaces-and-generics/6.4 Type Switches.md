When working with interfaces, we sometimes need to handle values differently based on their concrete runtime type rather than just the behaviour defined by the interface. In Go, this is done using **type switches**, which are similar to pattern matching on types in Java, allowing us to branch logic based on the underlying type of an interface value.

From 6.3, we can solve the issue of potential runtime errors with a type switch. To get the concrete type type for `p` in a type, we can write  `switch p.(type) { ... }`, or we can initialize the concrete type into a variable like `switch concreteValue := p.(type)` if you need to use the concrete value. Note that this only works if `p` is an interface value, type switches cannot be used on non-interface types.

`main.go`:
```go
package main

import "fmt"

type Printer interface {
	Print()
}

type Person struct {
	Name string
}

func (p *Person) Print() {
	fmt.Println("Name:", p.Name)
}

func callPrint(p Printer) {
	switch concreteValue := p.(type) {
	case *Person:
		if concreteValue == nil {
			fmt.Println("Cannot print: Person is nil")
			return
		}
		concreteValue.Print()
	default:
		p.Print()
	}
}

func main() {
	validPerson := &Person{Name: "Tyler"}
	callPrint(validPerson) // Works

	var nilPerson *Person = nil
	callPrint(nilPerson) // Works, now Handled safely
}
```
