Consider the program:

`main.go`:
```go
package main

func main() {}

func add(a, b any) any {
    aInt, aIsInt := a.(int)
    bInt, bIsInt := b.(int)

	if aIsInt && bIsInt {
		return aInt + bInt
	}
	
	aFloat, aIsFloat := a.(float64)
	bFloat, bIsFloat := b.(float64)
	
	if aIsFloat && bIsFloat {
		return aFloat + bFloat
	}
	
	aString, aIsString := a.(string)
	bString, bIsString := b.(string)
	
	if aIsString && bIsString {
		return aString + bString
	}
	
	return nil
}
```

Currently, if we try to do something like:

`main.go` (snippet):
```go
import "fmt"

func main() {
	result := add(1, 2)
	resultPlus1 := result + 1 // compile time error
	fmt.Println(resultPlus1)
}
```

This would not work, because the return type of `any` is too wide. We also don't want to do a type assertion to extract the type from `result`, we can do better than that here, we want Go to know that if we pass for example two integers, we get back an integer. We can solve these problems by narrowing the types down from `any` using **generics**. 

`main.go`
```go
package main

import "fmt"

func main() {
	result := add(1, 2)
	resultPlus1 := result + 1
	fmt.Println(resultPlus1)
}

func add[T int | float64 | string](a, b T) T {
	return a + b
}
```

> In `func add[T int | float64 | string](a, b T) T`, the square brackets declare a **type parameter** `T`, similar to generics in Java. Instead of using syntax like `T extends X`, Go restricts `T` using a **type constraint**, written as `int | float64 | string`, meaning T can be any one of those types.
