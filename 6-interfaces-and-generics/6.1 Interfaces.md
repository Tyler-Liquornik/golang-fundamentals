`todo.go`:

package todo

import (
	"encoding/json"
	"errors"
	"os"
)

type Todo struct {
	Text string `json:"text"`
}

func New(text string) (Todo, error) {
	if text == "" {
		return Todo{}, errors.New("text cannot be empty")
	}

	return Todo{
		text,
	}, nil
}

func (t Todo) String() string {
	return "Todo text: " + t.Text
}

func (t Todo) Save() error {
	fileName := "todo.json"
	filePathWithName := "6-interfaces-and-generics/6.1-6.2/" + fileName

	// Note: json.Marshal() will only marshal the exported fields of the struct,
	// so all fields have been made public.
	jsonBytes, err := json.Marshal(t)
	if err != nil {
		return err
	}
	return os.WriteFile(filePathWithName, jsonBytes, 0644)
}

A note and a todo share some behaviour we can capture in an interface, but may implement some of it differently. Let's pull out the improve the code by pulling out some common behaviour into an interface to reduce duplicate & near duplicate code. We seek to create a new function `saveData(data <Note || Todo>)` that can save either a `Todo` or a `Note`, so we do need an interface `Savable` that both implement.

`main.go` (snippet):
```go
type Saver interface {
	Save() error
}
```

> The convention for SAM interfaces (i.e., functional interfaces) in Go is to name them with the same name as the method plus an "er" at the end. So `Saver` here is more idiomatic than the more Java-like `Savable`, although this is common for non-SAM interfaces still.

Notice how interfaces can have methods in their definition, unlike structs which can only have receiver methods. There's also no `default` methods like Java, everything is implicitly `abstract`, and no `public static final` fields either.

import (
	"bufio"
	"fmt"

type Saver interface {
	Save() error
}

func main() {
	title, content := getNoteData()
	userNote, err := note.New(title, content)
	if err != nil {
		fmt.Println("Error creating note instance:", err)
		return

	todoString := userTodo.String()
	fmt.Println(todoString)

	saveData(userNote)
	saveData(userTodo)
}

func saveData(data Saver) {
	err := data.Save()
	if err != nil {
		fmt.Println("Error saving data:", err)
	}
	fmt.Println("Data saved successfully!")
}

func getTodoData() string {
	text := getUserInput("Enter todo text: ")
	return text
}

func getNoteData() (string, string) {
	title := getUserInput("Enter note title: ")
	content := getUserInput("Enter note content: ")
	return title, content
}

}

> A key Go feature is that types don’t explicitly declare that they implement an interface. Instead, any type that has the required methods automatically satisfies the interface at compile time, enabling loose coupling between types and abstractions.
