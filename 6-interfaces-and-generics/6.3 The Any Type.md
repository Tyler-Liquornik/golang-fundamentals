When working with interfaces, we sometimes need to handle values differently based on their concrete runtime type rather than just the behaviour defined by the interface. In Go, this is done using **type switches**, which are similar to pattern matching on types in Java, allowing us to branch logic based on the underlying type of an interface value.

`main.go`:
```go
package main

import "fmt"

type Printer interface {
	Print()
}

type Person struct {
	Name string
}

// Print compiles fine, but runtime panic if p==nil
func (p *Person) Print() {
	fmt.Println("Name:", p.Name)
}

func callPrint(p Printer) {
	p.Print()
}

func main() {
	validPerson := &Person{Name: "Tyler"}
	callPrint(validPerson) // Works

	var nilPerson *Person = nil
	callPrint(nilPerson) // Runtime panic
}
```

Be careful, as the `any` type is powerful but gives up some type safety, potentially causing runtime errors.
